# Reverting Changes

## Reverting a Commit

### `git revert HEAD`

Creates a new commit that undoes the latest commit.

```bash
$ git revert HEAD

[master a1b2c3d] Revert "Added login feature"
 1 file changed, 2 deletions(-)
```

---

### `git revert --no-edit HEAD`

Reverts the latest commit without opening an editor.

```bash
$ git revert --no-edit HEAD
```

---

### `git revert -n HEAD`

Applies the revert but does not create a commit. (brings back to staging)

```bash
$ git revert -n HEAD

$ git status

Changes to be committed:
    modified: app.py
```

---

## Reverting a Single File

### `git restore filename.extension`

Restores an unstaged file to its last committed state.

```bash
$ git restore app.py
```

---

### `git checkout -- filename.extension`

Older command for restoring a file.

```bash
$ git checkout -- app.py
```

---

## Unstaging Files

### `git restore --staged filename.extension`

Removes a file from the staging area.

```bash
$ git restore --staged app.py
```

---

### `git restore --staged .`

Removes all staged files from the staging area.

```bash
$ git restore --staged .
```
