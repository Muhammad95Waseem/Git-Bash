# Git Basics

## What is Git?
Git is a distributed version control system used to track changes in files and projects.

## What is Version Control?
Version control stores different versions of a project, allowing you to review, restore, and collaborate on changes.


`.git` is a hidden folder created by Git that stores commits, branches, configuration, and the complete history of the repository.

---

## Check Git Installation


### `git --version`


Displays the installed Git version.

```bash
$ git --version
git version 2.50.1
```

---

## Create a Repository

### `git init`

Initializes Git in the current directory.

```bash
$ git init
Initialized empty Git repository in /project/.git/
```

### `git init repo-name`

Creates a new directory and initializes Git inside it.

```bash
$ git init my-project
Initialized empty Git repository in /my-project/.git/
```

---

## Check Repository Status

### `git status`

Check Repository Status Shows tracked, untracked, and modified files.

```bash
$ git status
On branch master

No commits yet

nothing to commit
```

---

## Stage Files


### `git add file-name.extension`

Stages a specific file for the next commit.

```bash
$ git add app.py
```

### `git add .`

Stages all new and modified files in the current directory.

```bash
$ git add .
```

---

## Save Changes

### `git commit -m "message"`

Creates a commit with a descriptive message.

```bash
$ git commit -m "Added login feature"

[master (root-commit) 8f3e2c1] Added login feature
1 file changed, 20 insertions(+)
```
